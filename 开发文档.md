# 项目开发指令：《情绪缓冲卡》Web Demo

请基于 Vite + React + TypeScript + Tailwind CSS 开发一个 Web Demo。

项目名称：情绪缓冲卡

项目一句话：
当用户在短视频中刷到令人愤怒的内容时，系统弹出一张轻量卡片，引导用户通过 10 秒点击“怒气团子”完成情绪缓冲，再由 AI 将冲动情绪转化为理性表达。

当前版本不接入抖音、快手、小红书或任何真实短视频平台 SDK。
只需要在 Web 页面中模拟一个短视频竖屏窗口，完成可展示、可交互、可演示的产品 Demo。

---

## 一、最重要的产品原则

本项目不要做复杂功能。

最终只保留 4 个核心动作：

### STEP 1：用户刷到愤怒视频

模拟用户在短视频流中刷到令人愤怒的视频。

示例视频主题可以是虚构场景：

- 宠物被虐待
- 职场 PUA
- 老人被欺负
- 消费者维权无门
- 弱者被不公平对待

注意：
不要使用真实人物、真实机构、真实公共事件。
不要使用抖音 Logo 或高度相似的平台商标。
只做泛化短视频 UI。

---

### STEP 2：系统弹出情绪缓冲卡

当用户停留约 2 秒后，系统显示 AI 识别状态：

“AI 正在理解这条内容……”

然后弹出情绪缓冲卡。

卡片核心文案必须使用：

“这条确实让人火大。
先别急着开骂，捏一下？”

这句话是全产品最重要的一句话，必须突出显示。

卡片按钮：

“捏一捏，冷静 10 秒”

卡片副文案可选：

“给情绪一个出口，给表达一个更好的开始。”

卡片风格：
- 半透明毛玻璃
- 圆角
- 蓝紫渐变按钮
- 不要全屏遮挡
- 像短视频界面中的轻量浮层

---

### STEP 3：进入 10 秒互动

只保留一个互动方式：

用户疯狂点击「怒气团子」。

不要做枕头。
不要做沙包。
不要做复杂的多玩法。

原因：
枕头太传统，容易变成普通解压小游戏。
“怒气团子”更有视觉变化，也更能体现情绪可视化。

怒气团子的视觉变化路径：

初始状态：
“红温炸毛怪兽”

结束状态：
“软乎乎冷静团子”

交互要求：

用户每点击一次怒气团子：
- 团子产生弹性缩放动画
- 身上的红色怒气逐渐减少
- 炸毛程度逐渐降低
- 表情从暴躁变平静
- 周围的火焰 / 闪电逐渐消失
- 冷静进度逐渐上升
- 可以出现小气泡、小星星、降温波纹等反馈

倒计时：
10 秒。

页面要显示：
- 倒计时
- 冷静进度
- 点击次数
- 怒气团子的状态变化

示例状态变化：

0%：
红色、炸毛、皱眉、冒火、周围有闪电。

30%：
红色变浅，炸毛减少，表情仍不爽。

60%：
变成粉紫色，火焰减少，表情开始放松。

100%：
变成软乎乎团子，闭眼微笑，周围是气泡和柔光。

---

### STEP 4：AI 把火气转成理性表达

10 秒互动结束后，进入结果页。

页面先显示 AI 理解卡：

“AI 理解到：
你气的可能不是视频本身，
而是那种‘明明不公平，却没人负责’的感觉。”

然后展示 Before / After 对比。

Before 冲动评论：

“真是服了！太离谱了！
垃圾！无语！！！
气死我了！！！！”

After 理性表达：

请生成三种版本：

1. 克制版
“这件事最值得关注的，是后续处理是否公开透明。”

2. 有力版
“大家愤怒不是因为爱吵，而是因为类似问题一次次出现，却没人真正负责。”

3. 犀利但不攻击版
“最让人难受的不是事情发生，而是发生之后依然有人试图轻描淡写。”

用户可以点击切换三种版本。

结果页还要展示：

- 冷静完成度，例如 76%
- 情绪释放指数，例如 82
- 提示语：“冷静一下，表达更有力量。”
- 按钮：“再体验一次”

注意：
不要做“愤怒排行榜”。
如果需要排行，只能叫：
- 冷静力排行
- 情绪回收值
- 理性表达力

六、怒气团子组件要求
组件名建议：
AngerDumpling.tsx
它是整个 Demo 的核心视觉组件。
必须能根据 progress 变化展示不同状态。
type AngerDumplingProps = {
  progress: number; // 0-100
  clickCount: number;
  onClick: () => void;
};
视觉状态建议：
function getDumplingMood(progress: number) {
  if (progress < 30) return "furious";
  if (progress < 60) return "cooling";
  if (progress < 90) return "softening";
  return "calm";
}
不同状态对应：
furious：
-  红色 
-  炸毛 
-  皱眉 
-  冒火 
-  闪电 
cooling：
-  橙红色 
-  毛刺减少 
-  表情缓和 
softening：
-  粉紫色 
-  圆润 
-  火焰消失 
-  出现气泡 
calm：
-  淡紫 / 白色 
-  闭眼微笑 
-  软乎乎 
-  柔光 
-  小星星

七、互动计分逻辑
当前 Demo 不需要复杂心理评估，只做交互强度统计。
数据：
type BufferStats = {
  clickCount: number;
  duration: number;
  releaseScore: number;
  calmProgress: number;
};
建议计算：
const calmProgress = Math.min(100, clickCount * 4 + elapsedSeconds * 3);
const releaseScore = Math.min(100, clickCount * 3 + elapsedSeconds * 2);
注意：
 这些数值只代表互动强度，不代表真实心理诊断。
页面底部可以小字提示：
“本结果仅代表本次互动强度，不代表心理诊断。”
